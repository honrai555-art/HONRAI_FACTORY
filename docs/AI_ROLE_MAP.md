# HONRAI_FACTORY AI ROLE MAP

## 目的

HONRAI_FACTORY では複数の AI ツールを **役割分担** して使う。
このドキュメントは、どの AI に何を任せるかを定義する。

---

# 全体像

```
┌─────────────────────────────────────────────┐
│           戦略司令塔（設計・世界観）           │
│         ChatGPT  /  Claude                    │
└──────────────────┬──────────────────────────┘
                   │ 設計・指示
┌──────────────────▼──────────────────────────┐
│           実装参謀（コード・構造整理）          │
│         Cursor  /  VSCode                     │
└──────────────────┬──────────────────────────┘
                   │ 実装・修正
┌──────────────────▼──────────────────────────┐
│           製造兵站（自動生成・大量修正）        │
│         Codex  /  Docker  /  WSL2             │
└──────────────────┬──────────────────────────┘
                   │ 成果物
┌──────────────────▼──────────────────────────┐
│           物流倉庫（保管・同期）               │
│              GitHub                           │
└──────────────────┬──────────────────────────┘
                   │ 遠隔操作
┌──────────────────▼──────────────────────────┐
│           遠隔操作（スマホ・通知）             │
│    Chrome Remote Desktop  /  Discord Bot     │
└─────────────────────────────────────────────┘
```

# 戦略司令塔

## ChatGPT

| 項目 | 内容 |
|------|------|
| 主な用途 | 設計、戦略、世界観、キャリア設計、組織設計 |
| 得意 | ブレインストーミング、企画立案、文章生成 |
| 向かない | 大規模コードベースの直接編集 |

## Claude

| 項目 | 内容 |
|------|------|
| 主な用途 | 設計、戦略、長文分析、コードレビュー |
| 得意 | 構造化された設計、原因分析、ドキュメント執筆 |
| 参照 | `CLAUDE.md`（リポジトリ内ルール） |
| 向かない | ローカルファイルの直接操作（Claude Code 除く） |

### 共通ルール

* 設計・方針は **ここで決めてから** 実装参謀へ渡す
* 世界観・20タイプ・漫画金型は `AGENTS.md` を参照
* 大規模変更は事前に影響範囲を整理する

# 実装参謀

## Cursor

| 項目 | 内容 |
|------|------|
| 主な用途 | コード編集、AI 実装、構造整理、ファイル修正 |
| 得意 | リポジトリ全体を読みながらの最小限修正 |
| 参照 | `.cursorrules`（リポジトリ内ルール） |
| 環境 | Windows + PowerShell |

### 依頼の仕方

1. **何をしたいか** を一文で書く
2. **困っていること** があれば症状を書く
3. **大きな変更は避ける** と伝える

テンプレート: `prompts/`

## VSCode

| 項目 | 内容 |
|------|------|
| 主な用途 | 手動編集、デバッグ、拡張機能利用 |
| 得意 | 細かい手動調整、Git 操作、ターミナル実行 |

# 物流倉庫

## GitHub

| 項目 | 内容 |
|------|------|
| 主な用途 | コード保管、バージョン管理、AI 共有、自動同期 |
| ルール | `.env` は commit 禁止、main への force push 禁止 |
| 参照 | `workflow/github-basics.md` |

### AI への Git 操作ルール

* commit / push / PR は **ユーザーが明示依頼した場合のみ**
* 秘密情報が含まれていないか、commit 前に確認する

# 製造ライン別 AI 分担

## 漫画ライン

| 工程 | 担当 AI / ツール |
|------|------------------|
| 原案・脚本 | ChatGPT / Claude |
| GPT 漫画生成 | scripts/gpt（OpenAI API） |
| 画像生成 | ComfyUI / Stable Diffusion |
| 監視・通知 | Discord Webhook + Bot |
| コード修正 | Cursor |

## キャリアカードライン

| 工程 | 担当 AI / ツール |
|------|------------------|
| フォーム設計 | ChatGPT / Claude |
| 診断ロジック | GAS + OpenAI API |
| スライド生成 | Google Slides |
| データ管理 | Google Spreadsheet / Lark Base |

## メタバースライン

| 工程 | 担当 AI / ツール |
|------|------------------|
| 3D モデル | Blender |
| シーン構築 | Unity / Spatial |
| ビルド | scripts/unity_build.ps1 |
| 遠隔起動 | Discord Bot `!unity` |

# 20タイプと AI の使い分け

HONRAI 世界は 20 の基本タイプで構成される（`AGENTS.md` 参照）。

| フェーズ | 主担当 | 作業内容 |
|----------|--------|----------|
| 世界観・キャラ設計 | 戦略司令塔 | タイプ割り当て、設定文書 |
| 原案・ストーリーボード | 戦略司令塔 + 実装参謀 | `workspace/manga`, `workspace/storyboards` |
| 生成・ビルド | 製造兵站 | `scripts/`, `output/`, `builds/` |
| 保管・同期 | 物流倉庫 | GitHub |
| 確認・通知 | 遠隔操作 | Discord Bot |

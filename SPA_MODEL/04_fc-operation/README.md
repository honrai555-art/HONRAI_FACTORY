# 04_fc-operation — ファクトリー運用製造ライン

## 目的
HONRAI_FACTORYの製造ライン全体を運用するため、ログ確認、ビルド状況、Discord通知、GitHub整理、作業KPI集計を扱う製造ラインです。制作ラインが安定して再現可能に動く状態を保ちます。

## 入力
- `logs/` の操作履歴、ビルドログ、エラー、メトリクス
- `pipelines/` の実行定義
- `scripts/orchestrator.py` と各種運用スクリプト
- GitHub Issues / Pull Request / Actions の結果
- Discord通知に必要なイベント情報と環境変数

## 処理
1. 対象ライン、実行日時、入力データ、設定を確認する
2. ビルド、変換、通知、GitHub整理などの運用タスクを実行する
3. 結果を概要ログと詳細ログに分けて記録する
4. 重要な成功・失敗・完了イベントのみDiscordへ通知する
5. 不要な一時ファイルや大容量成果物をGit管理対象から外す

## 出力
- 運用ログ: `logs/`（Git管理対象外）
- 成果物: `builds/` または各ライン指定の出力先（Git管理対象外を基本とする）
- GitHub整理結果: commit / Pull Request / main反映状況
- Discord通知: 主要イベントの要約

## KPI
- ビルド成功率
- エラー検知から対応までの時間
- GitHub mainの整理完了数
- 不要ファイルのGit混入防止数
- Discord通知の主要イベント到達率

## AIへの指示
このフォルダは独立した製造ラインです。
変更する場合は以下を守ってください。

1. 既存ファイルを確認する
2. README.mdを確認する
3. 出力先を勝手に変更しない
4. 不要ファイルをGitに含めない
5. 変更後はgit statusを確認する
6. 問題なければcommitしてpushする

# 製造管理本部

## 役割
製造管理本部は、HONRAI_FACTORY の各製造ラインを Lark Base 中心で統合管理し、GitHub・Discord・各フォルダに分散していた進捗、依頼、納品、売上、成長DB更新状況を一元的に見える化する部署です。

- 全製造ラインの進捗管理
- 依頼管理
- 納品管理
- 売上管理
- 成長DB更新率管理

## 成長DB中心構造
- 目的: 成長DBを増やす
- 資産: 成長DB
- KPI: 更新率

## KPI
- 更新率
- 完了率
- 納期遵守率
- 案件数

## 管理対象
- `docs/`: 製造ライン組織図、Lark Base設計、ダッシュボード定義
- `templates/`: 週次・月次などのレポートテンプレート
- `reports/`: 実施後レポート、定例集計、意思決定ログ
- `dashboards/`: Lark Base / BI / 表示設計の補助資料

## 運用方針
1. 各製造ラインの案件・タスク・納品・売上・成長DB更新状況を Lark Base に集約する。
2. Lark Base の `PROJECTS`、`TASKS`、`GROWTH_DB`、`SALES` を正本として扱う。
3. GitHub はコード・ドキュメント履歴、Discord は通知・相談、Lark Base は管理台帳として役割を分ける。
4. 週次レポートで未完了、売上、更新率、翌週重点を確認する。

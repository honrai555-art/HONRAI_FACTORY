# Lark Base設計書

## 成長DB中心構造
- 目的: 成長DBを増やす
- 資産: 成長DB
- KPI: 更新率

## 設計方針
Lark Base は HONRAI_FACTORY の管理台帳の正本です。GitHub は履歴、Discord は通知、各フォルダは成果物保管、Lark Base は案件・タスク・成長DB・売上の統合管理に使います。

## PROJECTS

| フィールド | 型の目安 | 用途 |
|---|---|---|
| 案件名 | Text | 管理対象となる案件名。 |
| 担当部署 | Single select / Link | 主担当の製造ライン。 |
| 担当者 | User / Text | 人間または担当AI。 |
| 開始日 | Date | 案件開始日。 |
| 締切 | Date | 納期・完了予定日。 |
| 状態 | Single select | 未着手 / 進行中 / レビュー中 / 完了 / 保留。 |

## TASKS

| フィールド | 型の目安 | 用途 |
|---|---|---|
| タスク名 | Text | 作業単位の名称。 |
| 部署 | Single select / Link | タスクを担当する製造ライン。 |
| 担当 | User / Text | 人間または担当AI。 |
| 状態 | Single select | 未着手 / 進行中 / レビュー中 / 完了 / 保留。 |
| 期限 | Date | タスクの期限。 |

## GROWTH_DB

| フィールド | 型の目安 | 用途 |
|---|---|---|
| ユーザー名 | Text | 成長DB対象者名。 |
| 種 | Text / Select | 初期資質、芽になる要素。 |
| 花 | Text / Select | 表面化した強み、表現された成果。 |
| 実 | Text / Select | 納品・売上・成果として結実した内容。 |
| 更新日 | Date | 最終更新日。 |
| 冠位 | Select | 成長段階・ランク。 |

## SALES

| フィールド | 型の目安 | 用途 |
|---|---|---|
| 商品名 | Text | 販売した商品・サービス名。 |
| 金額 | Currency / Number | 売上金額。 |
| 購入者 | Text / Link | 購入者名または顧客レコード。 |
| 日付 | Date | 購入日・入金日。 |

## 推奨リレーション
- `PROJECTS.担当部署` → 製造ライン一覧の部署名
- `TASKS.部署` → 製造ライン一覧の部署名
- `TASKS` → `PROJECTS` へ案件リンクを追加すると進捗集計しやすい
- `GROWTH_DB.ユーザー名` → `SALES.購入者` と照合し、成長と売上の関係を見る

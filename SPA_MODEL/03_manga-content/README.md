# 03_manga-content — 漫画コンテンツ製造ライン

## 目的
HONRAI世界のキャラクター、クエスト、舞台をもとに、漫画原案、コマ割り、セリフ、ナレーション、ストーリーボード設計を生成する製造ラインです。感情のリズムと行動結果の連続性を保ちながら、制作可能な漫画素材へ落とし込みます。

## 入力
- `workspace/manga/` の漫画原案、ネーム、プロット
- `workspace/storyboards/` のページ設計、コマ割り、絵コンテ
- `SPA_MODEL/shared-data/` の世界観、タグ、タイトル、キャラクタールール
- 対象話数、ページ数、登場キャラクター、テーマ、演出意図

## 処理
1. 既存の原案、ストーリーボード、共有データを確認する
2. 話の目的、感情変化、クエスト結果を整理する
3. ページ単位でコマ割り、構図、セリフ、ナレーションを作成する
4. 必要な背景、道具、3D参照、AI画像生成指示を分離する
5. レビュー用MarkdownやJSONにまとめ、出力先へ保存する

## 出力
- 漫画原案: `workspace/manga/`
- ストーリーボード: `workspace/storyboards/`
- 制作用プロンプト: `SPA_MODEL/03_manga-content/output/` または指定成果物フォルダ
- プレビュー: `SPA_MODEL/03_manga-content/BuildPreviews/`（Git管理対象外）

## KPI
- 原案更新率
- 完成ページ数
- ストーリーボード完了数
- セリフ・ナレーションのレビュー通過数
- 重要事件、感情表現、テーマタグの反映率

## AIへの指示
このフォルダは独立した製造ラインです。
変更する場合は以下を守ってください。

1. 既存ファイルを確認する
2. README.mdを確認する
3. 出力先を勝手に変更しない
4. 不要ファイルをGitに含めない
5. 変更後はgit statusを確認する
6. 問題なければcommitしてpushする

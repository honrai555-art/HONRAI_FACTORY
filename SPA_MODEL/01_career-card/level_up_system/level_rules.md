# level_rules.md

## Ver3のレベル設計

Ver3のレベルアップは、固定ゴールではなく、ユーザーが設定したゴールの進捗に連動します。

ただし、共通レベルは全ユーザーで維持します。フリーランス、学習塾の生徒、講師、クリエイターのいずれでも、同じLv.1〜Lv.10を使い、ゴールカテゴリーに応じて条件の中身を読み替えます。

## 共通レベル

| レベル | 名称 | 共通条件 | ゴール連動の見方 |
|---:|---|---|---|
| Lv.1 | ゴールを設定した | user_goal と success_condition を設定した | 自分の次のステージを決めた |
| Lv.2 | 本来性を言語化した | childhood_dream、好きだったこと、自然にできることから honrai_theme を言語化した | ゴールと本来性の接続が見えた |
| Lv.3 | 最初の行動を書き込んだ | ゴールに向けた最初の action_log を保存した | 最初のキャリアブロックを得た |
| Lv.4 | 人に伝えた | ゴール、作品、学習計画、サービス、挑戦を誰かに伝えた | 外部接点が生まれた |
| Lv.5 | 仮説を立てた | 達成方法、学習計画、提供価値、作品設計などの仮説を立てた | 進め方が見えた |
| Lv.6 | 初回提供・初回提出・初回挑戦をした | ゴールに応じた初回アクションを外へ出した | 初回提出、初回受験、初回営業、初回公開など |
| Lv.7 | 成果が出た | success_condition に関係する成果が出た | 点数改善、顧客反応、売上、作品閲覧など |
| Lv.8 | 継続できた | 一定期間または複数回継続した | 学習習慣、継続案件、連続制作など |
| Lv.9 | 他者に教えられた | 自分の方法を他者に説明・指導できた | 教材化、共有、後輩支援、講座化など |
| Lv.10 | 相応しい次のステージに到達した | ユーザーが設定したゴールの success_condition を満たした | ゴール達成、または次ステージへ移行可能 |

## ゴールカテゴリー別の読み替え例

| ゴールカテゴリー | Lv.6の例 | Lv.7の例 | Lv.10の例 |
|---|---|---|---|
| 仕事化 | 初回提供 | 顧客反応または初回収益 | 相応しい仕事に近い提供が成立 |
| 学習習慣 | 初回提出・初回学習挑戦 | 連続学習または小テスト改善 | 学習習慣が定着 |
| 志望校 | 過去問初回挑戦 | 模試・過去問の改善 | 志望校合格または合格圏到達 |
| 資格 | 試験範囲の初回演習 | 模試合格ライン到達 | 資格取得 |
| 収益化 | 初回営業 | 初回収益 | 目標売上達成 |
| 講師化 | 初回体験会 | 受講生獲得 | 受講生目標達成 |
| 作品化 | 初回公開 | 反応・販売・応募 | ポートフォリオ完成または作品販売 |

## キャリアブロックとレベルの関係

キャリアブロックは、レベルアップの証拠になります。

| ブロック | 主な意味 | レベルへの使い方 |
|---|---|---|
| 分かったブロック | 気付き、理解、発見 | Lv.2、Lv.5の根拠 |
| できたブロック | 実践、行動、挑戦 | Lv.3、Lv.6の根拠 |
| 経験ブロック | 成果、習慣、継続 | Lv.7、Lv.8の根拠 |
| 仕事化ブロック | 販売、収益、顧客獲得 | フリーランスのLv.6〜Lv.10の根拠 |

学習塾向けでは、仕事化ブロックを「提出・挑戦・外部評価ブロック」として読み替えます。

## レベルアップ判定の流れ

1. user_goal と success_condition を確認する
2. 現在地、current_gap、goal_progress_rate を確認する
3. 行動入力を保存する
4. 入力内容からキャリアブロックを生成する
5. ゴール進捗率と進路エラー率を更新する
6. 共通レベル条件を満たすか判定する
7. current_level と current_rank を更新する
8. 次のゴール連動クエストを生成する
9. 成長DBへ保存する

## 冠位設計

冠位は、レベルとゴール進捗率を組み合わせて表示します。

| 冠位 | 目安 |
|---|---|
| ゴール見習い | Lv.1 |
| 本来性探索者 | Lv.2 |
| 初動者 | Lv.3 |
| 伝達者 | Lv.4 |
| 仮説設計者 | Lv.5 |
| 初回挑戦者 | Lv.6 |
| 成果獲得者 | Lv.7 |
| 継続者 | Lv.8 |
| 伝承者 | Lv.9 |
| 次ステージ到達者 | Lv.10 |

## 降格ではなくゴール修正

HONRAI LEVEL UP SYSTEM では、ユーザーを罰するための降格は行いません。

更新が止まった場合や進路エラー率が高い場合は、ゴールを見直し、次の小さなクエストへ再接続します。

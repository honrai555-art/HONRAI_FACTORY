# level_rules.md

## レベルアップ方針

HONRAI LEVEL UP SYSTEM Ver3 のレベルは、ユーザーが設定したゴールに近づくための行動とキャリアブロックの蓄積で上がります。

レベルアップは競争ではありません。ゴール、本来性、行動、成果がつながったことを祝う通知です。

## Lv.1〜Lv.10

| Lv | 名前 | 条件 | 主な証拠 |
|---:|---|---|---|
| 1 | ゴール発見 | ユーザーゴールを設定した | user_goal / goal_reason |
| 2 | 現在地確認 | current_gap と next_quest が出た | 診断結果 |
| 3 | 初回行動 | ゴール連動クエストを1回実行した | action_log |
| 4 | 気付き獲得 | 分かったブロックを獲得した | insight_log / career_block_summary |
| 5 | 実践継続 | できたブロックを複数回獲得した | action_log の継続 |
| 6 | 経験化 | 行動が習慣または成果になった | 経験ブロック / result_log |
| 7 | 進路修正 | route_error_rate を下げる修正ができた | goal修正 / next_quest修正 |
| 8 | 価値提供 | 他者反応、提出物、販売、合格判定などが出た | result_log |
| 9 | 仕事化・達成接続 | ゴール達成条件に大きく近づいた | success_condition の進捗 |
| 10 | 次ステージ解放 | ゴール達成または次のゴールを設定した | goal_status / 新user_goal |

## 冠位

冠位はレベル帯の呼び名です。

| レベル帯 | 冠位 | 意味 |
|---|---|---|
| Lv.1〜2 | 種火 | ゴールと現在地を言語化し始めた状態 |
| Lv.3〜4 | 点火 | 行動と気付きが生まれた状態 |
| Lv.5〜6 | 鍛造 | 実践と経験を積んでいる状態 |
| Lv.7〜8 | 進化 | 進路エラーを修正し価値提供へ向かう状態 |
| Lv.9〜10 | 解放 | ゴール達成または次ステージへ進む状態 |

## レベルアップ判定の流れ

1. 行動入力を保存する
2. ゴールとの接続を確認する
3. キャリアブロックを生成する
4. goal_progress_rate を更新する
5. route_error_rate を再計算する
6. Lv条件を満たしているか判定する
7. current_level / current_rank / level_up_reason を更新する
8. Discordまたはアプリ内で通知する

## ゴール別の判定例

### フリーランス

```text
ゴール：月10万円の副業収益
行動：自分のサービスを1人に説明した
獲得ブロック：できたブロック
判定：Lv.3 初回行動、またはLv.5 実践継続
```

### 学習塾の生徒

```text
ゴール：学習習慣の定着
行動：7日連続で英単語を学習した
獲得ブロック：経験ブロック
判定：Lv.6 経験化
```

### クリエイター

```text
ゴール：作品販売
行動：作品ページの説明文を書き、1人に見せた
獲得ブロック：できたブロック / 経験ブロック
判定：Lv.5 実践継続、またはLv.8 価値提供
```

## 進路エラー時の扱い

route_error_rate が高くなっても降格しません。

進路エラーは、ゴールや行動を修正するためのサインです。必要な場合は `goal_status` を「修正中」にして、goal_reason、desired_state、success_condition、next_quest を見直します。

## 既存ルールから継承する考え方

- レベルアップ理由を必ず残す
- 小さな行動も成長DBの資産として扱う
- 仕事化ゴールでは `workization_status` も評価する
- 結果が出ない期間も、学習や気付きがあればキャリアブロックとして評価する

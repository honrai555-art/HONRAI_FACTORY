# screen_design.md

## 画面設計の目的

画面設計の目的は、ユーザーが自分で設定したゴールに向かって、今日やること、進捗、キャリアブロック、レベルアップを迷わず確認できるようにすることです。

## ホーム画面

### 表示項目

- ユーザー名
- ユーザーの現在ゴール
- ゴールカテゴリー
- ゴールステータス
- ゴール進捗率
- 本来性一致スコア
- 進路エラー率
- 現在レベル
- 現在冠位
- キャリアブロック数
- 次のクエスト
- 今日の行動入力ボタン
- ゴール修正ボタン

### 画面メッセージ例

```text
今のゴール：月10万円の副業収益
進捗：35%
次のクエスト：自分のサービスを1人に説明する
獲得できるブロック：できたブロック
```

## ゴール設定画面

### 入力項目

- 子供の頃の夢、好きだったこと
- 現在地
- 相応しい次のステージ
- ゴールカテゴリー
- ゴール理由
- 望む状態
- 達成条件
- 期限

### 保存後の処理

1. honrai_alignment_score を算出する
2. current_gap を生成する
3. next_quest を生成する
4. goal_status を設定済みまたは実行中にする
5. Lv.1 ゴール発見を判定する

## 行動入力画面

### 入力項目

- 今日やったこと
- 気付き
- 成果
- ゴールへの進捗
- 次にやりたいこと

### 保存後の処理

1. career_block_type を判定する
2. career_block_summary を生成する
3. goal_progress_rate を更新する
4. route_error_rate を再計算する
5. current_level を判定する
6. level_up_reason を生成する

## レベルアップ画面

### 表示項目

- 上がったレベル
- 新しい冠位
- レベルアップ理由
- 獲得したキャリアブロック
- ゴール進捗への影響
- 次のクエスト

### 表示例

```text
Lv.5 実践継続へレベルアップ！
理由：自分のサービスを1人に説明し、できたブロックを獲得しました。
ゴール「月10万円の副業収益」へ一歩近づいています。
```

## 進路エラー修正画面

進路エラー率が高い場合、責める表現ではなく修正提案として表示します。

### 表示項目

- 現在のゴール
- 本来性とのズレの可能性
- 軽くできる次のクエスト
- ゴール理由の見直し提案
- ゴール修正ボタン

## 管理画面

### 表示項目

- 登録ユーザー数
- 更新ユーザー数
- 更新率
- ゴールカテゴリー別ユーザー数
- 生成キャリアブロック数
- レベルアップ数
- Discord通知数
- エラー数

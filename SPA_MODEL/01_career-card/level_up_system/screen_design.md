# screen_design.md

## 画面設計の目的

画面設計では、ユーザーに成長DBを見せるのではなく、「自分で設定したゴールに向かってレベルアップし続けるゲーム」を体験させます。

中心UIは以下です。

1. ゴール設定画面
2. ホーム画面
3. 行動入力画面
4. キャリアブロック画面
5. レベルアップ画面
6. ゴール進捗画面

## 1. ゴール設定画面

### 目的

ユーザーが現在地と本来性を踏まえて、相応しい次のステージを設定する。

### 入力項目

- ゴール名
- ゴールカテゴリー
- ゴール理由
- 望む状態
- 期限
- 達成条件
- 現在のギャップ

### 保存時の処理

1. honrai_alignment_score を算出する
2. current_gap を整理する
3. next_quest を生成する
4. goal_status を「設定済み」または「実行中」にする

## 2. ホーム画面

### 目的

ユーザーの現在ゴールとゲーム状態を一目で分かるようにする。

### 表示項目

- ユーザーの現在ゴール
- ゴールカテゴリー
- ゴール進捗率
- 本来性スコア
- 進路エラー率
- 現在レベル
- 現在冠位
- キャリアブロック数
- 次のクエスト
- 今日の行動入力ボタン

### UIイメージ

```text
HONRAI LEVEL UP

現在ゴール：月10万円の副業収益
カテゴリー：収益化
ゴール進捗率：35%
本来性スコア：78
進路エラー率：22%
現在レベル：Lv.5
現在冠位：仮説設計者
キャリアブロック数：14
次のクエスト：自分のサービスを1人に説明する

[今日の行動を入力する]
[キャリアブロックを見る]
[ゴールを見直す]
```

## 3. 行動入力画面

### 目的

今日の行動を保存し、ゴール進捗とキャリアブロックを更新する。

### 入力項目

- 今日やったこと
- 気付き
- 成果
- ゴールへの進捗

### 保存時の処理

1. 入力内容を action_log / result_log に保存する
2. 入力からキャリアブロック種類を判定する
3. キャリアブロックを生成する
4. goal_progress_rate を更新する
5. route_error_rate を更新する
6. レベルアップ条件を確認する
7. 次のゴール連動クエストを生成する

## 4. キャリアブロック画面

### 目的

獲得したブロックをテトリスのように積み上げ、ゴールに近づいている実感を出す。

### 表示項目

- 分かったブロック
- できたブロック
- 経験ブロック
- 仕事化ブロック
- 対応ゴール
- ゴール進捗への影響

## 5. レベルアップ画面

### 目的

ゴール進捗とキャリアブロックが条件を満たしたとき、レベルアップと冠位上昇を祝う。

### 表示項目

- 新レベル
- 新冠位
- 対応ゴール
- ゴール進捗率
- レベルアップ理由
- 獲得ブロック
- 次のクエスト

## 6. ゴール進捗画面

### 目的

ゴール達成条件、現在地、ギャップ、次の行動を見える化する。

### 表示項目

- ゴール名
- ゴールカテゴリー
- 達成条件
- 期限
- 現在地
- 現在のギャップ
- ゴール進捗率
- 本来性スコア
- 進路エラー率
- 次のクエスト

## 学習塾向け表示

学習塾向けには、仕事化ではなく学習継続を前面に出します。

- 現在ゴール：志望校合格、学習習慣化、資格取得など
- 表示する価値：学習習慣化、継続率向上、進捗可視化、保護者説明
- ブロック表現：提出ブロック、挑戦ブロック、継続ブロックなどへの読み替え可

## フリーランス向け表示

フリーランス向けには、仕事化・収益化を前面に出します。

- 現在ゴール：初回顧客獲得、月10万円達成、継続案件獲得など
- 表示する価値：仕事化、収益化、本来性発見、実績づくり
- ブロック表現：仕事化ブロック、顧客獲得ブロック、収益化ブロック

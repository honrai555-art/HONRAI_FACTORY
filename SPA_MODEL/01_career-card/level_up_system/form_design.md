# form_design.md

## フォーム設計の目的

このフォームは、ユーザーが HONRAI LEVEL UP SYSTEM Ver3 に自分の現在地とゴールを書き込み、成長DBを更新するための入口です。

ユーザーが記入した内容から、進路エラー診断、ゴール連動クエスト、キャリアブロック生成、レベルアップ判定を行い、結果を成長DBへ蓄積します。

## Ver3の最優先方針

Ver3 では「ユーザーが設定する相応しい次のステージ」を最優先にします。

- ゴール（相応しい次のステージ）の**最終決定者はユーザー**です。
- 診断提案やメンター提案は**補助**であり、ユーザーの決定を上書きしません。
- どこからゴール候補が来たか（ユーザー／診断／メンター）は `next_stage_source` で記録し、最終的に採用したステージは常にユーザーが確定します。

## 必須入力項目

フォームの必須項目は以下の10項目です。各項目は成長DB（`growth_db_schema.md`）のカラムに対応します。

| No | フォーム項目名 | 入力例 | 型 | 必須 | 説明 | 対応する成長DBカラム |
|---:|---|---|---|:---:|---|---|
| 1 | ユーザー名 | 空野ミカ | string | Yes | 表示名。 | `user_name` |
| 2 | 今の状態 | 副業を始めたいが何から手を付けるか迷っている | text | Yes | ユーザーの現在地。今おかれている状況や悩み。 | `current_state` |
| 3 | 子供の頃の夢 | 漫画家 | text | Yes | 子供の頃の夢、好きだったこと、自然にできること。本来性の起点。 | `childhood_dream` |
| 4 | 相応しい次のステージ | 月10万円の副業収益 | text | Yes | ユーザー自身が決める「相応しい次のステージ」。Ver3の最優先項目で、最終決定者はユーザー。 | `next_stage` / `user_goal` |
| 5 | next_stage_source | user | enum | Yes | 次のステージ候補の出どころ。`user`（自分で決めた）/ `diagnosis`（診断提案を採用）/ `mentor`（メンター提案を採用）。提案由来でも最終確定はユーザーが行う。 | `next_stage_source` |
| 6 | 進路エラータイプ | 行動停滞型 | enum | Yes | 今ズレている種類。下記「進路エラータイプ」から選ぶ。失敗ではなく修正サイン。 | `route_error_type` |
| 7 | 今日の行動 | 自分のサービスを友人1人に説明した | text | Yes | 今日やったこと。キャリアブロック生成の素材。 | `action_log` |
| 8 | 気づき | 物語づくりは相手の経験を引き出せると分かった | text | Yes | 行動から得た気付き・理解・発見。 | `insight_log` |
| 9 | キャリアブロック | できたブロック：サービスを1人に説明できた | text | Yes | 今日獲得した成長単位。種類（分かった／できた／経験／仕事化）と要約。 | `career_block_type` / `career_block_summary` |
| 10 | 次のクエスト | 説明文を1枚にまとめて別の1人に見せる | text | Yes | 次に行う小さな行動。ゴールへ近づく一歩。 | `next_quest` |

## next_stage_source の値

| 値 | 意味 | 最終決定者 |
|---|---|---|
| user | ユーザーが自分で設定した | ユーザー |
| diagnosis | 進路エラー診断の提案を採用した | ユーザー（採用判断） |
| mentor | メンター／講師の提案を採用した | ユーザー（採用判断） |

`diagnosis` と `mentor` は補助です。提案をそのまま強制せず、ユーザーが「相応しい次のステージ」として確定したものを採用します。

## 進路エラータイプ

進路エラータイプは、`route_error_rate`（数値）を補足する分類です。責める分類ではなく、どの修正が必要かを示します。

| タイプ | 説明 | 主な修正アクション |
|---|---|---|
| ズレなし | ゴールと行動が一致している | 行動継続、記録強化 |
| 目標未設定型 | 相応しい次のステージがまだ決まっていない | next_stage を設定する |
| 本来性ズレ型 | ゴールが子供の頃の夢・好きなことと離れている | goal_reason / next_stage を見直す |
| 行動停滞型 | ゴールはあるが行動が止まっている | next_quest を軽くして1行動だけ実行 |
| 過負荷型 | 目標や行動が大きすぎて続かない | success_condition を小さく分解する |
| 方向転換型 | ゴール自体を変えたほうがよい | goal_status を「修正中」にして再設定 |

## 入力フロー

1. ユーザー名、今の状態、子供の頃の夢を入力する
2. 相応しい次のステージを入力する（必要なら診断・メンター提案を参考にするが、確定はユーザー）
3. next_stage_source を選ぶ
4. 進路エラータイプを選ぶ
5. 今日の行動、気づきを入力する
6. キャリアブロックを入力する（または行動・気づきから自動生成する）
7. 次のクエストを入力する（または自動生成する）

## 保存後の処理

1. `next_stage` を `user_goal` として成長DBへ保存する（Ver3の正式ゴール）
2. `next_stage_source` を保存し、最終決定者がユーザーであることを記録する
3. `route_error_type` と `route_error_rate` を更新する
4. `career_block_type` / `career_block_summary` を確定する
5. `goal_progress_rate` を更新する
6. `current_level` / `current_rank` / `level_up_reason` を判定する
7. レベルアップ・ゴール達成時は通知する（`notification_flow.md` 参照）

## バリデーション

- 必須10項目が未入力の場合は保存しない
- `next_stage_source` は `user` / `diagnosis` / `mentor` のいずれか
- `route_error_type` は「進路エラータイプ」の一覧から選ぶ
- 自由記述項目（今の状態、今日の行動、気づき など）は空文字を許可しない

# HONRAI LEVEL UP SYSTEM

## このシステムの目的

HONRAI LEVEL UP SYSTEM は、ユーザーが「子供の頃の夢を仕事にできない」という進路エラーに気づき、本来の自分に還り続けながら、相応しい仕事までレベルアップし続けるための成長ゲーム環境です。

このシステムは単なる記録フォームではありません。以下の循環を継続的に回し、成長DBを増やし、更新率を上げることを最優先にします。

```text
進路エラー通知
↓
スキル・学習提示
↓
行動書き込み
↓
レベルアップ通知
↓
成長DBに蓄積
↓
AI工場で再活用
```

HONRAI における本システムは、「本来の自分に還り続けることで、相応しい仕事を手に入れるための成長OS」の中核となるレベルアップラインです。

## 顧客の問題

顧客の根本問題は、子供の頃の夢をそのまま仕事にできず、現在の仕事・生活・学習が本来のテーマからズレていることです。

よくある状態は以下です。

- 子供の頃にやりたかったことを忘れている
- 現在の仕事が生活のためだけになっている
- 自分に相応しい仕事の仮説を持てていない
- 学ぶべきスキルが分からない
- 行動した記録が残らず、成長を実感できない
- 成果や失敗がAI工場で再活用されない

## 解決策

解決策は、ユーザーが本来の自分に還り続けるための通知・学習・行動・記録・再活用の仕組みを作ることです。

HONRAI LEVEL UP SYSTEM は、以下を提供します。

1. **進路エラー通知**：本来の自分からズレている可能性を通知する
2. **スキル・学習提示**：相応しい仕事に近づく次のスキルを提示する
3. **行動書き込み**：今日の小さな行動を成長DBへ記録する
4. **レベルアップ通知**：本来性レベル・冠位・仕事化段階の上昇を祝う
5. **成長DB蓄積**：進化の記録を資産化する
6. **AI工場で再活用**：漫画原案、キャリアカード、通知、ストーリー、3D表現へ変換する

## 成長DBの定義

成長DBとは、「本来の自分に還り、相応しい仕事を手に入れるまでの進化の記録」です。

保存対象は、プロフィール情報だけではありません。夢、テーマ、進路エラー、次に学ぶスキル、行動ログ、結果ログ、仕事化段階、レベルアップ理由を継続的に蓄積します。

成長DBは、HONRAI_FACTORY における資産です。AI工場はこのDBを使い、ユーザーごとの物語、キャリア提案、通知文、学習クエスト、漫画金型、3D表現の素材へ変換します。

## 更新率をKPIにする理由

HONRAI の事業目的は「成長DBを増やす」ことです。そのため、KPIは単なる登録数ではなく、更新率を最重要指標にします。

更新率を重視する理由は以下です。

- ユーザーが本来の自分に還り続けているかを測れる
- 行動ログと結果ログが増えるほど、AI工場で再活用できる素材が増える
- レベルアップ通知の精度が上がる
- 相応しい仕事への仮説検証が進む
- DBが一度きりのアンケートではなく、成長OSとして機能する

更新率の基本定義は以下です。

```text
更新率 = 対象期間内に action_log / result_log / current_level / workization_status のいずれかを更新したユーザー数 ÷ 登録ユーザー数
```

## 進路エラー通知の流れ

1. childhood_dream と current_job を比較する
2. honrai_theme と現在の行動ログを比較する
3. route_error_type を判定する
4. 本来の自分からズレている可能性を通知する
5. ユーザーに「違和感」「取り戻したい夢」「小さな再接続行動」を書き込んでもらう

通知の目的は不安を煽ることではなく、本来のテーマへの再接続です。

## スキル・学習提示の流れ

1. desired_job と current_level を確認する
2. 現在の honrai_score と workization_status を確認する
3. 次に必要な next_skill を1つに絞る
4. learning_task を小さな学習クエストとして提示する
5. 完了後に action_log へ記録してもらう

学習提示では、ユーザーがすぐ動けるように、抽象的な助言ではなく「今日できる最小行動」に変換します。

## 行動書き込みの流れ

1. 毎日または主要イベント後に行動書き込み通知を送る
2. ユーザーが今日の行動、感情、結果、次の一歩を記録する
3. action_log と result_log に保存する
4. 更新日時 updated_at を更新する
5. レベルアップ条件に該当するかを判定する

行動書き込みは、更新率を上げるための最重要接点です。

## レベルアップ通知の流れ

1. action_log / result_log / workization_status を確認する
2. level_rules.md の条件に照らして current_level を判定する
3. レベルが上がった場合、level_up_reason を生成する
4. 本来性レベル・冠位・仕事化段階の上昇を通知する
5. 次の学習クエストを提示する

レベルアップ通知は、継続意欲を生む報酬です。通知文は、ユーザーの夢と言葉を反映して生成します。

## 成長DBへの保存項目

成長DBには最低限、以下を保存します。

- user_id
- user_name
- childhood_dream
- honrai_theme
- route_error_type
- current_job
- desired_job
- current_level
- honrai_score
- next_skill
- learning_task
- action_log
- result_log
- workization_status
- level_up_reason
- updated_at

詳細な項目定義は `growth_db_schema.md` を参照します。

## 今後の自動化方針

今後は以下の順で自動化します。

1. Lark / Googleフォームから行動ログを入力できるようにする
2. Slack / ChatGPT / Discord で通知を送る
3. action_log と result_log からレベルアップを自動判定する
4. honrai_score をAIが算出する
5. desired_job に応じて next_skill と learning_task を自動生成する
6. 成長DBを漫画原案、キャリアカード、クエスト、3Dシーンへ変換する
7. 更新率ダッシュボードを作り、成長DB増加を可視化する
8. 冠位、称号、クエスト、協力プレイを加え、成長ゲームとして拡張する

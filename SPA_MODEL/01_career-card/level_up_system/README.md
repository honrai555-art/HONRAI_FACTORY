# HONRAI LEVEL UP SYSTEM Ver3

## 目的

HONRAI LEVEL UP SYSTEM は、ユーザー自身が設定した「相応しい次のステージ」へ向かって、進路エラーを修正しながら、キャリアブロックを積み上げ、レベルアップし続けるWebサービスです。

Ver3では、運営がゴールを「相応しい仕事」に固定しません。フリーランス、学習塾の生徒、講師、クリエイターでは到達したい場所が異なるため、ユーザーごとにゴールを設定し、そのゴールへ向かうゲーム体験を作ります。

## サービス定義

HONRAIとは、ユーザー自身が設定したゴールに向かって、進路エラーを修正しながら、キャリアブロックを積み上げ、レベルアップし続けるWebサービスです。

## HONRAIの共通OS

HONRAIの共通OSは、以下の4要素で構成されます。

```text
進路エラー診断
+
キャリアブロック
+
レベルアップシステム
+
ユーザーゴール設計
```

この共通OSにより、ゴールが「月10万円の副業収益」でも「志望校合格」でも「学習習慣化」でも、同じ仕組みで現在地、ズレ、次のクエスト、行動記録、レベルアップを扱えます。

## Ver3の設計思想

### 旧設計

```text
HONRAIのゴール = 相応しい仕事
```

### 新設計

```text
HONRAIのゴール = ユーザーが設定する「相応しい次のステージ」
```

HONRAIは、ユーザーごとに異なるゴールへ向かってレベルアップし続けるWebサービスです。

学習塾の生徒にとってのゴールは、学習習慣・志望校・資格・夢です。

フリーランスにとってのゴールは、仕事化・収益化・相応しい仕事です。

そのため、HONRAIはゴールを固定せず、ユーザーが設定したゴールに対して、進路エラー診断・クエスト・キャリアブロック・レベルアップを提供します。

## なぜユーザーゴール設計に変更するか

ユーザー属性ごとに「相応しい次のステージ」が異なるためです。

| ユーザー種別 | ゴール例 |
|---|---|
| フリーランス | 相応しい仕事、月10万円達成、初回顧客獲得、継続案件獲得 |
| 学習塾の生徒 | 学習習慣の定着、志望校合格、資格取得、将来の夢発見、苦手科目克服 |
| 講師 | 受講生10名獲得、講座販売、指導実績づくり |
| クリエイター | 作品販売、ポートフォリオ完成、案件獲得 |

## 新しいゲームループ

```text
ユーザーゴール設定
↓
進路エラー診断
↓
ゴール連動クエスト
↓
行動記録
↓
キャリアブロック生成
↓
レベルアップ
↓
冠位上昇
↓
ゴール進捗上昇
↓
新しいクエスト
↓
行動記録
（循環）
```

このループにより、ユーザーは「DBを更新する」のではなく、「自分で設定したゴールに近づくために行動を記録する」状態になります。

## キャリアブロックの定義

キャリアブロックとは、成長DBの最小単位です。

行動入力から生成される小さな成長単位であり、ゴール進捗、レベルアップ、冠位上昇、AI工場での再活用の材料になります。

## キャリアブロックの種類

| 種類 | 意味 | 例 |
|---|---|---|
| 分かったブロック | 気付き、理解、発見 | 勉強は朝の方が続くと分かった |
| できたブロック | 実践、行動、挑戦 | 自分のサービスを1人に説明できた |
| 経験ブロック | 成果、習慣、継続 | 7日連続で英単語を学習できた |
| 仕事化ブロック | 販売、収益、顧客獲得 | 初回相談で報酬を受け取った |

## ユーザーゴール設計

ユーザーゴール設計は、ユーザーが自分の現在地と本来性を踏まえて、相応しい次のステージを設定するための設計です。

ゴールには以下を持たせます。

- ゴール名
- ゴールカテゴリー
- ゴール理由
- 望む状態
- 期限
- 達成条件
- 本来性一致スコア
- 現在のギャップ
- 次のクエスト
- 更新日時

詳細は `goal_design.md` を参照します。

## 進路エラー診断の修正

Ver3の進路エラー診断は、「相応しい仕事」とのズレだけを診断しません。

ユーザーが設定したゴールと、本来性とのズレを診断します。

診断結果では以下を出します。

- 現在地
- 本来性スコア
- ゴール一致率
- 進路エラー率
- 次のクエスト

## 本来性スコアの修正

本来性スコアは、「ユーザーが設定したゴール」と「子供の頃の夢・好きだったこと・自然にできること」の一致率として扱います。

たとえば、学習塾の生徒が「志望校合格」をゴールにした場合でも、そのゴールが本人の好きなこと、将来の夢、自然に続けやすい学び方とどれだけ合っているかを評価します。

## クエスト設計

クエストは、ユーザーゴールに応じて自動生成されます。

クエストには必ず以下を含めます。

- クエスト名
- 対応するゴール
- 今日やる行動
- 獲得できるキャリアブロック
- ゴール進捗への影響
- レベルアップ条件

### クエスト例

```text
ゴール：月10万円の副業収益
クエスト：自分のサービスを1人に説明する
仕事化への接続：見込み客獲得の練習
獲得ブロック：できたブロック
ゴール進捗への影響：営業行動の初回経験が増える
レベルアップ条件：人に伝えた / 初回提供に近づく
```

## ホーム画面の表示項目

ホーム画面では、ユーザーが設定したゴールと現在のゲーム状態を表示します。

- ユーザーの現在ゴール
- ゴールカテゴリー
- ゴール進捗率
- 本来性スコア
- 進路エラー率
- 現在レベル
- 現在冠位
- キャリアブロック数
- 次のクエスト
- 今日の行動入力ボタン

## 行動入力画面の入力項目

行動入力画面は、ゴール進捗とキャリアブロック生成の入口です。

入力項目は以下です。

- 今日やったこと
- 気付き
- 成果
- ゴールへの進捗

保存時に、入力内容からキャリアブロックを生成し、ゴール進捗率、進路エラー率、レベル条件を更新します。

## 学習塾向け表現

学習塾向けには、HONRAIを「相応しい仕事」ではなく、生徒が自分のゴールに向かって継続学習できる仕組みとして表現します。

学習塾向けの価値は以下です。

- 学習習慣化
- 継続率向上
- 生徒の目標設定
- 進捗可視化
- 保護者説明しやすい
- 差別化できる

## フリーランス向け表現

フリーランス向けには、HONRAIを「相応しい仕事までレベルアップする仕組み」として表現します。

フリーランス向けの価値は以下です。

- 仕事化
- 収益化
- 本来性発見
- 行動継続
- 実績づくり
- キャリアブロック蓄積

## 成長DBの位置付け

成長DBは、ユーザーが見る主役ではなく、裏側で蓄積される資産です。

保存内容は以下です。

- ユーザーゴール
- ゴールカテゴリー
- ゴール理由
- 達成条件
- 期限
- 現在のギャップ
- ゴール進捗率
- 本来性一致スコア
- ゴールステータス
- キャリアブロック
- 行動履歴
- 成果履歴
- 仕事化履歴
- 冠位履歴

## 事業目的とKPI

事業目的は、成長DBを増やすことです。

KPIは更新率です。

更新率の基本定義は以下です。

```text
更新率 = 対象期間内にゴール更新、行動記録、キャリアブロック生成、ゴール進捗更新のいずれかを行ったユーザー数 ÷ 登録ユーザー数
```

## 最終ゴール

ユーザーが自分でゴールを設定し、そのゴールに向かって更新し続けることで、キャリアブロックが増え、レベルが上がり、ゴールに近づき、その記録が成長DBとして蓄積される状態を作ります。

その成長DBをAI工場で再活用し、FC加盟店が継続学習システムとして提供できるようにします。

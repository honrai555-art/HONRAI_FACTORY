# notification_flow.md

## 通知フローの目的

通知フローは、ユーザーが本来の自分に還り続け、相応しい仕事へ近づくための行動を止めない仕組みです。通知のゴールは、成長DBの更新率を上げることです。

## 通知全体の循環

```text
1. 進路エラー通知
   ↓
2. スキル・学習通知
   ↓
3. 行動書き込み通知
   ↓
4. レベルアップ通知
   ↓
成長DB更新・AI工場で再活用
```

## 1. 進路エラー通知

### 目的

本来の自分からズレている可能性を通知し、子供の頃の夢と現在の仕事・行動を再接続する。

### 発火条件

- childhood_dream と current_job の接続が弱い
- action_log が一定期間更新されていない
- honrai_score が低下した
- route_error_type が未判定
- ユーザーが「今の仕事に違和感がある」と記録した

### 通知内容

- 子供の頃の夢の再提示
- 現在のズレの可能性
- route_error_type の仮判定
- 今日できる小さな再接続行動
- 行動書き込みへの誘導

### 通知テンプレート

```text
【進路エラー通知】
{user_name}さん、子供の頃の夢「{childhood_dream}」と現在の状態「{current_job}」の間に、少しズレが出ている可能性があります。

これは失敗ではなく、本来の自分へ戻るサインです。
今日の小さな再接続行動：{learning_task}

1行でよいので、感じた違和感や行動を書き込んでください。
```

## 2. スキル・学習通知

### 目的

相応しい仕事へ近づくために、次に学ぶべきスキルを提示する。

### 発火条件

- desired_job が設定された
- current_level が更新された
- action_log はあるが result_log が弱い
- workization_status を次段階へ進めたい
- ユーザーが「次に何を学ぶべきか分からない」と記録した

### 通知内容

- desired_job の再提示
- next_skill の提示
- learning_task の提示
- 完了後の記録依頼

### 通知テンプレート

```text
【スキル・学習通知】
{user_name}さんの相応しい仕事の仮説は「{desired_job}」です。
次に伸ばすスキルは「{next_skill}」。

今日の学習クエスト：{learning_task}
完了したら、何を学び、何を試したかを action_log に書き込んでください。
```

## 3. 行動書き込み通知

### 目的

今日の行動記録を促し、成長DBの更新率を上げる。

### 発火条件

- 毎日の定時
- 学習クエスト提示から一定時間後
- ユーザーが成果物を作った
- 顧客提供や相談対応が発生した
- updated_at が一定期間古い

### 通知内容

- 今日の行動の確認
- 感情・気づきの記録
- result_log への誘導
- レベルアップ判定の予告

### 通知テンプレート

```text
【行動書き込み通知】
{user_name}さん、今日の本来性ログを1行だけ残しましょう。

書くこと：
1. 今日やったこと
2. 気づいたこと
3. 次に試すこと

この書き込みが、成長DBを増やし、次のレベルアップ判定につながります。
```

## 4. レベルアップ通知

### 目的

本来性レベル・冠位・仕事化段階の上昇を通知し、継続意欲を高める。

### 発火条件

- level_rules.md の条件を満たした
- current_level が上昇した
- workization_status が進んだ
- honrai_score が一定以上上がった
- 顧客提供、収益化、継続提供などの成果が記録された

### 通知内容

- 新しい current_level
- 冠位または称号
- workization_status の変化
- level_up_reason
- 次のクエスト

### 通知テンプレート

```text
【レベルアップ通知】
{user_name}さん、レベルアップしました。

新レベル：Lv.{current_level}
仕事化段階：{workization_status}
理由：{level_up_reason}

子供の頃の夢「{childhood_dream}」は、相応しい仕事「{desired_job}」へ近づいています。
次のクエスト：{learning_task}
```

## 通知設計の原則

- 通知は責めるものではなく、本来の自分へ戻る合図にする
- 1通知1行動に絞る
- 更新率を上げるため、必ず書き込み導線を入れる
- ユーザーの言葉を引用し、AIっぽい一般論にしない
- レベルアップ通知は小さな進化も祝う

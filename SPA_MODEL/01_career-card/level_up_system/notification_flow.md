# notification_flow.md

## 通知方針

HONRAI LEVEL UP SYSTEM Ver3 の通知は、ユーザーが自分で設定したゴールに向かって行動し続けるためのきっかけです。

Discordは製造指示の主役ではなく、主要イベントの通知センターとして扱います。アプリ内通知、メール、Lark、Discordなどの実装先は分離できますが、通知内容は共通化します。

## 通知の種類

1. ゴール設定通知
2. 進路エラー診断通知
3. ゴール連動クエスト通知
4. 行動書き込み通知
5. キャリアブロック獲得通知
6. レベルアップ通知
7. ゴール達成通知
8. AI工場再活用通知

## 1. ゴール設定通知

### 目的

ユーザーが自分の「相応しい次のステージ」を設定したことを確認し、最初のクエストへつなげる。

### テンプレート

```text
【ゴール設定完了】
{user_name}さんの現在ゴールは「{user_goal}」です。
ゴールカテゴリー：{goal_category}
本来性一致スコア：{honrai_alignment_score}
最初のクエスト：{next_quest}
```

## 2. 進路エラー診断通知

### 目的

ゴール、本来性、現在行動のズレを責めずに知らせ、修正行動へ誘導する。

### テンプレート

```text
【進路エラー診断】
{user_name}さん、現在ゴール「{user_goal}」と今の行動の間に、少しズレが出ている可能性があります。
これは失敗ではなく、次のステージへ進むための調整サインです。
進路エラー率：{route_error_rate}%
今日の小さな修正クエスト：{next_quest}
```

## 3. ゴール連動クエスト通知

### 目的

今日やることを明確にし、行動入力へつなげる。

### テンプレート

```text
【今日のクエスト】
ゴール「{user_goal}」に近づく今日の行動です。
クエスト：{next_quest}
獲得できるブロック：{career_block_type}
完了したら、今日やったことと気付きを1行で記録してください。
```

## 4. 行動書き込み通知

### 目的

更新率を上げ、成長DBへ行動を残す。

### テンプレート

```text
【行動書き込み】
{user_name}さん、今日の行動を成長DBに残しましょう。
入力すること：今日やったこと / 気付き / 成果 / ゴールへの進捗
小さな1行でもキャリアブロックになります。
```

## 5. キャリアブロック獲得通知

### 目的

行動から生成された小さな成長単位を可視化する。

### テンプレート

```text
【キャリアブロック獲得】
{career_block_type} を獲得しました。
内容：{career_block_summary}
ゴール進捗：{goal_progress_rate}%
この記録はAI工場で教材、漫画、キャリアカード、3D表現の素材として再活用できます。
```

## 6. レベルアップ通知

### 目的

ユーザーが自分のゴールに近づいていることを祝う。

### テンプレート

```text
【レベルアップ】
{user_name}さん、Lv.{current_level}「{current_rank}」へレベルアップしました！
理由：{level_up_reason}
現在ゴール：{user_goal}
次のクエスト：{next_quest}
```

## 7. ゴール達成通知

### 目的

success_condition を満たしたことを祝い、次のステージ設定へ誘導する。

### テンプレート

```text
【ゴール達成】
{user_name}さん、ゴール「{user_goal}」の達成条件を満たしました。
積み上がったキャリアブロック：{career_block_count}個
次は、新しい「相応しい次のステージ」を設定できます。
```

## 8. AI工場再活用通知

### 目的

成長DBがAI漫画、教材、ゲーム、3Dオブジェクト、KAIDOWALKへ再活用されたことを知らせる。

### テンプレート

```text
【AI工場再活用】
{user_name}さんの成長DBから、新しい素材が生成されました。
生成物：{artifact_type}
保存先：{artifact_path}
確認用プレビュー：{preview_path}
```

## 通知頻度ルール

- ゴール設定、レベルアップ、ゴール達成は即時通知する
- 行動書き込みリマインドは1日1回までにする
- Discord通知は主要イベントに限定し、スパム化を避ける
- エラー通知にはプロジェクト名、実行時間、成果物の場所、スタックトレース概要を含める

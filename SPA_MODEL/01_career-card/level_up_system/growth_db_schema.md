# growth_db_schema.md

## 成長DBの目的

成長DBは、本来の自分に還り、相応しい仕事を手に入れるまでの進化の記録です。HONRAI LEVEL UP SYSTEM では、このDBを増やし、更新率を上げることを最優先にします。

## 項目定義

| 項目名 | 型 | 必須 | 説明 | 更新タイミング |
|---|---|---:|---|---|
| user_id | string | Yes | ユーザーを一意に識別するID。外部フォームや通知システムと連携する主キー。 | 初回登録時 |
| user_name | string | Yes | ユーザー名、表示名、またはニックネーム。 | 初回登録時／変更時 |
| childhood_dream | string | Yes | 子供の頃の夢。例：漫画家、先生、冒険家。 | 初回診断時／夢の再発見時 |
| honrai_theme | string | Yes | 本来の自分に還るための中心テーマ。HONRAI世界の「テーマ／モチーフ」に相当する。 | 診断後／言語化更新時 |
| route_error_type | string | Yes | 本来のテーマからのズレの種類。例：夢忘却型、生活優先型、他者期待型、行動停止型。 | 進路エラー診断時 |
| current_job | string | Yes | 現在の仕事、役割、学習状態。未就業の場合は現在の主活動を記録する。 | 初回登録時／状態変化時 |
| desired_job | string | Yes | 相応しい仕事の仮説。子供の頃の夢を現代的・仕事化可能な形へ翻訳したもの。 | 診断後／仮説更新時 |
| current_level | integer | Yes | Lv.1〜Lv.10 の本来性レベル。level_rules.md に基づいて判定する。 | 行動・結果更新時 |
| honrai_score | integer | Yes | 本来性スコア。0〜100で、夢・テーマ・行動・仕事化の一致度を表す。 | 診断時／定期更新時 |
| next_skill | string | Yes | 次に学ぶべきスキル。更新率を上げるため、原則1つに絞る。 | 学習提示時 |
| learning_task | string | Yes | next_skill を具体化した小さな学習クエスト。今日実行可能な粒度にする。 | 学習提示時 |
| action_log | text | Yes | 今日の行動、挑戦、学習、顧客提供などの記録。 | 毎日／主要イベント後 |
| result_log | text | Yes | 行動の結果、気づき、成果、失敗、顧客反応の記録。 | 行動後／振り返り時 |
| workization_status | string | Yes | 仕事化段階。例：未着手、仮説、試作、提供、収益化、継続提供、相応しい仕事。 | 仕事化の進展時 |
| level_up_reason | text | Yes | レベルアップした理由。ユーザーの行動・結果・本来性の変化を要約する。 | レベル更新時 |
| updated_at | datetime | Yes | 最終更新日時。更新率KPIの算出に使う。ISO 8601形式を推奨。 | すべての更新時 |

## route_error_type の例

| 種類 | 説明 |
|---|---|
| 夢忘却型 | 子供の頃の夢を忘れ、現在の仕事との接続が切れている |
| 生活優先型 | 生活維持を優先し、本来のテーマを後回しにしている |
| 他者期待型 | 親、学校、会社、社会の期待を優先しすぎている |
| 行動停止型 | やりたいことは分かっているが、行動ログが蓄積されていない |
| 仕事化未翻訳型 | 夢はあるが、現代の仕事や提供価値へ翻訳できていない |

## workization_status の例

| 段階 | 説明 |
|---|---|
| 未着手 | 夢やテーマはあるが行動がない |
| 仮説 | 相応しい仕事の仮説を立てた |
| 試作 | 小さな制作物、講座、企画、提案を作った |
| 提供 | 顧客や他者に提供した |
| 収益化 | 金銭、報酬、仕事依頼が発生した |
| 継続提供 | 継続的に提供できている |
| 相応しい仕事 | 本来性と収益・役割が一致している |

## 更新率に関わる主要項目

更新率を上げるため、特に以下の項目は短いサイクルで更新します。

- action_log
- result_log
- current_level
- honrai_score
- next_skill
- learning_task
- workization_status
- updated_at

## AI工場での再活用

成長DBは、以下のAI工場ラインで再活用します。

- キャリアカード生成
- 進路エラー診断
- レベルアップ通知文生成
- 学習クエスト生成
- 漫画原案・コマ割り・感情リズム化
- 3D世界観・舞台・ガジェット化
- 顧客向け提案書・サービス設計

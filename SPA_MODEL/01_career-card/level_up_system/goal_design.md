# goal_design.md

## ゴール設計の目的

ゴール設計の目的は、ユーザーが自分の現在地と本来性を踏まえて、相応しい次のステージを設定することです。

HONRAIでは、運営がゴールを固定しません。ユーザーごとに「相応しい次のステージ」を設定し、そのゴールに対して進路エラー診断、クエスト生成、キャリアブロック生成、レベルアップを行います。

## ゴール設計の基本方針

- ゴールはユーザー自身が設定する
- ゴールは後から修正できる
- ゴールは本来性と照合する
- ゴールは今日のクエストへ分解する
- ゴール進捗はキャリアブロックで増える
- ゴール達成後は、次のステージを再設定する

## ゴール項目

| 項目名 | 型 | 必須 | 説明 |
|---|---|---:|---|
| user_id | string | Yes | ゴールを設定したユーザーID。 |
| goal_title | string | Yes | ユーザーが設定した現在ゴール。例：月10万円の副業収益、志望校合格。 |
| goal_category | string | Yes | ゴールカテゴリー。仕事化、学習習慣、志望校など。 |
| goal_reason | text | Yes | なぜそのゴールを設定するのか。本人の動機。 |
| desired_state | text | Yes | ゴール達成時にどうなっていたいか。 |
| deadline | date | No | 達成期限。未定でもよい。 |
| success_condition | text | Yes | 達成判定条件。数値、提出物、習慣、状態など。 |
| honrai_alignment_score | integer | Yes | ゴールと本来性の一致率。0〜100。 |
| current_gap | text | Yes | 現在地とゴールの差分。 |
| next_quest | text | Yes | ゴールへ近づく次の行動。 |
| updated_at | datetime | Yes | ゴール情報の最終更新日時。 |

## ゴールカテゴリー

| カテゴリー | 説明 | 例 |
|---|---|---|
| 仕事化 | 本来性を仕事に接続する | 初回顧客獲得、継続案件獲得 |
| 学習習慣 | 学習を継続できる状態にする | 毎日30分学習、宿題提出率改善 |
| 志望校 | 志望校合格へ向かう | 第一志望校合格、模試偏差値アップ |
| 資格 | 資格取得へ向かう | 英検、簿記、IT資格 |
| 収益化 | 売上や報酬を得る | 月10万円、副業初収益 |
| 講師化 | 教える側になる | 受講生10名、講座販売 |
| 作品化 | 作品やポートフォリオを完成する | 漫画販売、作品集公開 |
| 起業 | 事業仮説やサービスを作る | MVP公開、初回契約 |
| 自己理解 | 本来性や将来像を言語化する | 将来の夢発見、得意の発見 |
| その他 | 上記に当てはまらない個別ゴール | 家族との約束、地域活動 |

## goal_status

| ステータス | 説明 |
|---|---|
| 未設定 | ゴールがまだない |
| 設定済み | ゴールはあるが行動前 |
| 実行中 | クエストと行動記録が進んでいる |
| 達成 | success_condition を満たした |
| 修正中 | ゴールを見直している |
| 中止 | ユーザー判断で停止した |

## ゴール設定フロー

1. 子供の頃の夢、好きだったこと、自然にできることを聞く
2. 現在地を確認する
3. ユーザーが望む次のステージを言語化する
4. goal_category を選ぶ
5. success_condition を決める
6. deadline を設定する
7. honrai_alignment_score を算出する
8. current_gap を出す
9. next_quest を生成する
10. goal_status を「設定済み」または「実行中」にする

## ゴール別クエスト生成の考え方

クエストは、ゴールから逆算して今日の行動へ分解します。

| ゴール例 | クエスト例 | 獲得ブロック |
|---|---|---|
| 月10万円の副業収益 | 自分のサービスを1人に説明する | できたブロック |
| 学習習慣の定着 | 今日の学習開始時間を決めて30分実行する | できたブロック |
| 志望校合格 | 苦手単元を1つ解き直す | 経験ブロック |
| 受講生10名獲得 | 講座の説明文を1つ作る | できたブロック |
| 作品販売 | 作品ページの説明文を書く | できたブロック |
| 将来の夢発見 | 好きだったことを10個書き出す | 分かったブロック |

## ゴール修正の考え方

ゴールが本来性とズレている場合、失敗ではなく進路エラーとして扱います。

進路エラー率が高い場合は、以下を行います。

- goal_reason を見直す
- desired_state を見直す
- success_condition を小さくする
- next_quest を軽くする
- goal_status を「修正中」にする

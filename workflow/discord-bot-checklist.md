# Discord bot 公開前チェックリスト

本番環境や一般ユーザーに Bot を公開する前に確認してください。

## セキュリティ

- [ ] Bot トークンは `.env` のみ（リポジトリに含まれていない）
- [ ] `.env` が `.gitignore` に入っている
- [ ] 必要最小限の **Intents** だけ有効化している
- [ ] 管理者権限を Bot に付与していない（本当に必要な場合のみ）

## Discord 開発者ポータル

- [ ] Application / Bot が作成済み
- [ ] **MESSAGE CONTENT INTENT** 等、必要な Intent を有効化
- [ ] OAuth2 URL で Bot をサーバーに招待済み
- [ ] 開発用サーバーと本番用サーバーを分けている（推奨）

## コード

- [ ] スラッシュコマンドが意図どおり登録されている
- [ ] エラー時に Bot が落ちず、ログに理由が残る
- [ ] レート制限を考慮（連続 API 呼び出しを避ける）
- [ ] 大規模変更なく、レビュー可能な diff になっている

## 運用

- [ ] `README` または bot フォルダに起動手順がある
- [ ] GitHub に push 済み（正本がクラウドにある）
- [ ] トークン漏洩時の **再生成手順** を把握している

## 漏洩したとき

1. Discord Developer Portal で **Bot トークンを Reset**
2. 新トークンを `.env` にのみ反映
3. Git 履歴にトークンが残っていないか確認（残っている場合は履歴から削除が必要）

AI に依頼する場合は `prompts/discord-bot-bugfix.md` を使い、**原因説明を先に** 求めてください。
